# Plume — 產品需求文件（PRD）

## 目標使用者

| 角色 | 需求 | 痛點 | 期望效益 |
|------|------|------|----------|
| 日常寫作者（主要：開發者本人） | 隨手寫筆記、草稿，邊寫邊看排版 | 現有工具太肥（Electron 系吃記憶體）或不順手（Web 版要開瀏覽器） | 秒開、輕量、寫完即存 |
| 長文作者 | 部落格/長文寫作後貼到發布平台 | 寫作工具與發布平台的渲染差異 | GFM 一致渲染 + 匯出 HTML |
| 技術文件讀者 | 檢查 README 等文件的渲染效果 | GitHub 上看要先 push | 本機直接開 `.md` 即看 |
| 語法測試者 | 快速驗證某段 Markdown 怎麼渲染 | 開線上工具太繞 | 開 app 貼上即所見 |

## 使用者故事

### US-1 即時預覽（P0）

作為寫作者，我想要打字時立刻看到渲染結果，以便邊寫邊確認排版。

驗收條件：
- Given 應用已開啟 → When 在編輯區輸入 GFM 語法（表格、任務清單、刪除線、程式碼區塊） → Then 預覽區在 200ms 內更新為正確渲染結果
- Given 使用注音/拼音輸入法 → When 組字過程中 → Then 編輯區組字不中斷、不跳字

### US-2 開檔與存檔（P0）

作為使用者，我想要開啟本機 `.md` 檔、編輯後存回去，以便把它當成日常編輯器。

驗收條件：
- Given 應用已開啟 → When Cmd+O 選擇 `.md` 檔 → Then 內容載入編輯區、視窗標題顯示檔名
- Given 已開啟的檔案被修改 → When Cmd+S → Then 寫回原路徑、dirty 標記消失
- Given 未命名新文件 → When Cmd+S → Then 跳出另存對話框
- Given 文件有未儲存變更 → When 關閉視窗 → Then 跳出確認對話框（儲存/放棄/取消）

### US-3 安全預覽他人文件（P0）

作為技術文件讀者，我想要安全地開啟別人寫的 `.md`，以便不被惡意內容攻擊。

驗收條件：
- Given 開啟含 `<script>` 或 `<img onerror=...>` 的 `.md` → When 預覽渲染 → Then 惡意碼被移除不執行，其餘內容正常顯示

### US-4 同步捲動（P1）

作為長文作者，我想要捲動編輯區時預覽區跟著捲，以便對照當前段落。

驗收條件：
- Given 文件長度超過一屏 → When 捲動編輯區 → Then 預覽區按比例跟隨至對應位置

### US-5 匯出 HTML（P1）

作為長文作者，我想要把渲染結果匯出成獨立 HTML，以便貼到發布平台或分享。

驗收條件：
- Given 編輯區有內容 → When 執行匯出 HTML → Then 產出自帶樣式的單一 `.html` 檔，瀏覽器開啟後渲染與預覽區一致

### US-6 最近開啟檔案（P1）

作為日常使用者，我想要快速重開最近編輯過的檔案，以便接續寫作。

驗收條件：
- Given 曾開啟過檔案 → When 查看最近檔案清單 → Then 列出最近 10 筆、點擊即開啟
- Given 應用重新啟動 → When 點擊最近檔案 → Then 仍可直接開啟（權限跨 session 保留）
- Given 清單中的檔案已被刪除 → When 點擊它 → Then 顯示非阻斷錯誤提示並從清單移除

### US-7 冊（Codex）檔案管理（P2）

作為日常寫作者，我想要把一個資料夾開成「冊」、在側邊欄瀏覽其中的 `.md` 檔案樹並切換不同的冊，以便管理多個寫作專案，不必逐一 Cmd+O。

驗收條件：
- Given 選定資料夾 → When 開啟為「冊」 → Then 側邊欄樹狀列出該資料夾（含子目錄）內所有 `.md`，點擊即開啟編輯
- Given 開過多個冊 → When 切換冊 → Then 側邊欄更新為該冊的檔案樹，授權跨 session 保留
- Given 進入撰態（沉浸寫） → When 切換 → Then 側邊欄自動隱藏，不破壞「落筆即沉浸」
- Given 在撰態（側欄被沉浸規則隱藏）→ When 開啟冊（選單／＋鈕／冊鈕）→ Then 自動切到「參」態並顯示側邊欄（開冊＝瀏覽檔案＝離開沉浸，避免「點了沒反應」）
- 註：新增／刪除／重命名交由系統檔案總管處理（L1 範圍不含 app 內檔案管理）

|------|------|------|--------|

| F-05 | 同步捲動 | 編輯區 → 預覽區單向比例捲動 | P1 |
| F-06 | 匯出 HTML | 自帶樣式的獨立 HTML 檔 | P1 |
| F-07 | 最近檔案 | 最近 10 筆，跨 session 有效 | P1 |
| F-08 | 外部連結外開 | 預覽區點擊 http(s) 連結用系統瀏覽器開啟，不在 app 內導航 | P1 |

| B-03 | 多分頁 | 同時編輯多個文件 | Backlog |
| B-04 | 超大檔優化 | >1MB 文件的增量渲染（屆時再評估 Rust 端解析） | Backlog |

## 非功能需求

| 類別 | 指標 |
|------|------|
| 效能 | 冷啟動 < 1.5s；輸入到預覽更新 < 200ms（含 debounce）；100KB 文件單次渲染 < 50ms |
| 資源 | App bundle < 15MB；閒置記憶體 < 150MB |
| 安全 | 所有渲染輸出過 DOMPurify；CSP 啟用；fs 權限最小化（僅 dialog 授權路徑） |
| 相容 | macOS 13+；中文 IME（注音/拼音）輸入正常 |
| 可靠 | 寫檔失敗不丟失編輯內容；渲染例外不影響編輯區 |
